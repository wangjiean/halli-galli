## ❌ 未完成/有问题的功能

### 1. Socket.IO 连接初始化 (🔴 严重问题)

**问题描述**:
- 用户进入大厅时，Socket 连接可能没有正确初始化
- 原因：`LobbyView.vue` 的 `onMounted` 中尝试连接 Socket，但此时 `authStore.token` 可能为空
- 导致：点击"创建房间"按钮没有反应（Socket 未连接，emit 无效）

**当前代码**:
```javascript
// LobbyView.vue onMounted
onMounted(() => {
  if (!socketStore.socket) {
    socketStore.connect(authStore.token);  // ❌ token 可能为 null
  }
  refreshRooms();
});
```

**需要实现**:
1. 在用户登录后立即建立 Socket 连接
2. 或者在访问需要 Socket 的页面前确保已连接
3. 添加连接状态检查和错误处理

**建议修复位置**:
- `HomeView.vue` - 在用户进入主页时建立连接
- 或在 `router.beforeEach` 中检查并连接

## 🔧 立即需要修复的问题

### P0 - 阻塞性问题

#### 1. Socket.IO 连接初始化

**文件**: `client/src/views/LobbyView.vue`, `client/src/views/HomeView.vue`

**修复方案**:

方案 A - 在主页建立连接：
```javascript
// HomeView.vue
onMounted(() => {
  if (!socketStore.socket) {
    socketStore.connect(authStore.token);
  }
});
```

方案 B - 在登录后建立连接：
```javascript
// auth.js
async function login(username, password) {
  const response = await api.post('/auth/login', { username, password });
  if (response.success) {
    token.value = response.token;
    user.value = response.user;
    localStorage.setItem('token', response.token);
    localStorage.setItem('user', JSON.stringify(response.user));
    
    // 新增：自动连接 Socket
    const socketStore = useSocketStore();
    socketStore.connect(response.token);
  }
  return response;
}
```

方案 C - 在 router 守卫中检查并连接：
```javascript
// router/index.js
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore();
  const socketStore = useSocketStore();
  
  if (to.meta.requiresAuth && !authStore.isAuthenticated) {
    next('/login');
  } else if (to.meta.requiresAuth && authStore.isAuthenticated && !socketStore.connected) {
    // 需要认证且已登录但 Socket 未连接
    socketStore.connect(authStore.token);
    next();
  } else if ((to.name === 'Login' || to.name === 'Register') && authStore.isAuthenticated) {
    next('/');
  } else {
    next();
  }
});
```

**推荐**: 方案 B + 方案 C 结合
