# 俄罗斯方块双人对战游戏 - 需求文档

## 项目概述

重构现有的俄罗斯方块游戏，实现一个支持局域网多人对战的 Web 游戏。用户可以通过浏览器访问游戏，登录账号后进行单人模式或双人对战模式的游戏。

---

## 一、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端 | Node.js + Express + Socket.IO | HTTP 服务 + WebSocket 实时通信 |
| 前端 | Vue 3 + Vite + Pinia | 组件化 UI + 状态管理 |
| UI 库 | Naive UI | 现代化组件库 |
| 渲染 | HTML5 Canvas | 游戏核心渲染 |
| 数据存储 | JSON 文件 (lowdb) | 用户账号、最高分持久化 |
| 认证 | JWT (jsonwebtoken) | 登录认证 |

## 二、功能需求

### 2.1 用户认证系统

#### 功能描述
- 所有用户访问游戏必须登录
- 支持账号注册和登录
- 登录状态持久化（7 天有效期）
- 支持多设备同时登录不同账号

#### 详细需求

**注册功能**
- 用户名：3-20 字符，字母数字下划线
- 密码：6-20 字符
- 密码使用 bcrypt 哈希存储
- 用户名唯一性校验

**登录功能**
- 用户名 + 密码验证
- 成功后返回 JWT token
- Token 存储于 localStorage
- Token 有效期 7 天

**路由守卫**
- 未登录用户访问任何页面重定向到登录页
- 登录页已登录用户重定向到主页

---

### 2.2 单人模式

#### 功能描述
经典的俄罗斯方块单人游戏，记录并保存最高分。

#### 详细需求

**游戏功能**
- 标准 7 种方块（I, J, L, O, S, T, Z）
- SRS 旋转系统 + Wall Kick
- 7-Bag 随机发生器
- 消行计分规则：
  - Single (1 行): 100 × level
  - Double (2 行): 300 × level
  - Triple (3 行): 500 × level
  - Tetris (4 行): 800 × level
  - Soft Drop: 1 分/格
  - Hard Drop: 2 分/格

**操作**
- 左/右箭头：移动
- 上箭头：旋转
- 下箭头：软降
- 空格：硬降
- ESC：暂停/继续
- P：暂停/继续

**UI 元素**
- 游戏画板（10×20）
- 下一个方块预览（1-3 个）
- 当前分数
- 当前等级
- 消行数
- 暂停覆盖层
- 游戏结束覆盖层（显示最终分数）

**最高分记录**
- 游戏结束时自动提交分数
- 后端比较并更新用户最高分
- 主页可查看所有玩家排行榜

---

### 2.3 双人对战模式

#### 功能描述
两个玩家实时对战，支持限时模式和淘汰模式。

#### 详细需求

**房间系统**
- 创建房间：设置房间名、模式、时间限制、是否开启攻击
- 加入房间：从大厅选择房间加入
- 房间状态：等待中、游戏中、已满
- 准备机制：双方都准备后开始倒计时（3 秒）
- 离开房间：房主离开则房间解散
- 超时清理：创建后 5 分钟无人加入则自动销毁

**对战模式**
1. **限时模式**
   - 时间到后比较分数
   - 分数高者获胜
   - 平分则加时赛

2. **淘汰模式**
   - 先堆积到顶部者输
   - 另一方获胜

**攻击机制（可选）**
- 消 2 行 → 给对手发 1 行垃圾
- 消 3 行 → 给对手发 2 行垃圾
- 消 4 行 → 给对手发 4 行垃圾
- 垃圾行从底部推入，随机留一个空格

**游戏同步**
- 客户端权威架构
- 每次操作后发送棋盘快照到服务器
- 服务器转发给对手
- 定期（500ms）广播完整状态作为兜底
- 对手棋盘实时渲染

**UI 元素**
- 左右分屏显示双方棋盘
- 中间信息栏（双方分数、等级、倒计时）
- 准备按钮
- 游戏结束覆盖层（显示胜负）

**断线处理**
- 玩家断线后保留房间 2 分钟
- 重连后恢复游戏状态
- 超时未重连则判负

---

### 2.4 排行榜

#### 功能描述
显示所有玩家的单人模式最高分排名。

#### 详细需求
- 按最高分降序排列
- 显示前 50 名
- 显示用户名、最高分、游戏场次
- 主页可访问

---

### 2.5 主页

#### 功能描述
游戏主界面，提供模式选择和导航。

#### 详细需求
- 显示当前登录用户
- 单人模式入口
- 双人对战入口（进入大厅）
- 排行榜入口
- 登出按钮

## 三、非功能需求

### 3.1 性能
- 游戏帧率 ≥ 60 FPS
- 双人模式网络延迟 < 100ms（局域网）
- 首屏加载时间 < 2 秒

### 3.2 兼容性
- 支持 Chrome、Firefox、Safari、Edge 最新两个版本
- 支持桌面和平板设备
- 最低分辨率 1280×720

### 3.3 安全性
- 密码 bcrypt 哈希存储
- JWT token 验证
- 简单的输入验证（家庭使用，不考虑复杂攻击）

### 3.4 部署
- 一键启动：`node server/index.js`
- 监听 0.0.0.0，局域网可访问
- 默认端口：7778

## 五、API 接口

### 5.1 认证接口

#### POST /api/auth/register
注册新用户
```json
// Request
{ "username": "player1", "password": "123456" }

// Response
{ "success": true, "user": { "id": "...", "username": "player1" } }
```

#### POST /api/auth/login
登录
```json
// Request
{ "username": "player1", "password": "123456" }

// Response
{ "success": true, "token": "jwt...", "user": { ... } }
```

### 5.2 排行榜接口

#### GET /api/leaderboard
获取排行榜
```json
// Response
{
  "success": true,
  "leaderboard": [
    { "rank": 1, "username": "player1", "highScore": 12000, "gamesPlayed": 25 }
  ]
}
```

### 5.3 分数接口

#### POST /api/scores
提交分数（单人模式结束）
```json
// Request (Header: Authorization: Bearer token)
{ "score": 12000 }

// Response
{ "success": true, "isNewRecord": true }
```

## 六、Socket.IO 事件

### 6.1 房间事件

#### Client → Server
- `room:create` - 创建房间
- `room:join` - 加入房间
- `room:leave` - 离开房间
- `room:ready` - 准备/取消准备
- `room:start` - 开始游戏（仅房主）

#### Server → Client
- `room:created` - 房间创建成功
- `room:joined` - 加入房间成功
- `room:left` - 玩家离开房间
- `room:updated` - 房间状态更新
- `room:countdown` - 开始倒计时

### 6.2 游戏事件

#### Client → Server
- `game:state` - 发送游戏状态快照
- `game:attack` - 发送攻击（消行）

#### Server → Client
- `game:opponent-state` - 对手状态
- `game:attack-received` - 收到攻击
- `game:over` - 游戏结束
