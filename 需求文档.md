## 🎯 项目概述

### 项目简介

**Halli Galli 德国心脏病** 是一款经典的反应类卡牌游戏，玩家需要快速识别卡牌图案并按铃抢牌。本项目实现局域网多人对战版本，支持经典和极限两种游戏模式。

### 核心特性

- 🎮 **双人对战**：实时竞技，考验反应速度
- 🍌 **经典模式**：60 张牌，1 种按铃条件
- 🐵 **极限模式**：72 张牌，5 种按铃条件
- 📱 **响应式设计**：支持桌面和移动设备
- 🔔 **实时同步**：WebSocket 双向通信

### 目标用户

- 卡牌游戏爱好者
- 反应速度挑战者
- 局域网聚会玩家

## 🎲 游戏规则

### 游戏模式

#### 经典模式 (Classic)

**牌组构成**：
- 60 张水果牌
- 4 种水果：🍌 香蕉、🍓 草莓、🍋 柠檬、🟣 李子
- 每种水果 1-5 张，共 20 张（4 × 5 = 20）
- 每张牌上有 1-5 个该水果图标

**按铃条件**：
- ✅ **5 张同种水果**：中央牌堆最上方 5 张牌是同一种水果

**示例**：
```
🍌🍌🍌🍌🍌  ← 可以按铃
🍓🍓🍓🍓🍓  ← 可以按铃
🍌🍋🍌🍋🍌  ← 不能按铃（混合）
```

#### 极限模式 (Extreme)

**牌组构成**：
- 64 张水果牌（同经典模式）
- 8 张动物牌
  - 🐵 猴子 × 3
  - 🐘 大象 × 3
  - 🐷 猪 × 2

**按铃条件**（5 种）：

1. **5 张同种水果**（同经典模式）

2. **5 张动物牌**
   ```
   🐵🐘🐷🐵🐘  ← 可以按铃
   🐵🍌🐘🐷🐵  ← 不能按铃（混入水果）
   ```

3. **4 种不同**
   ```
   🍌🍋🍓🐵  ← 4 种不同，可以按铃
   🍌🍋🍌🍓  ← 只有 3 种，不能按铃
   ```

4. **2 对牌**
   ```
   🍌🍌🍋🍋  ← 2 对，可以按铃
   🍌🍌🍌🍋  ← 只有 1 对，不能按铃
   ```

5. **连续数字**
   ```
   🍌₂ 🍋₃ 🍓₄  ← 2-3-4 连续，可以按铃
   🍌₂ 🍋₄ 🍓₅  ← 2-4-5 不连续，不能按铃
   ```

### 游戏流程

#### 1. 初始阶段
- 每位玩家获得 30 张（经典）或 36 张（极限）牌
- 牌面朝下放在自己面前
- 中央牌堆为空

#### 2. 出牌阶段
- 玩家轮流翻牌
- 将牌面朝上放到中央牌堆
- 出牌后回合交给下一位玩家

#### 3. 按铃阶段
- 任何玩家发现满足按铃条件时可按铃
- **正确按铃**：赢得中央牌堆所有牌
- **错误按铃**：失去 1 张牌给每位其他玩家

#### 4. 胜利条件
- 赢得所有牌的玩家获胜
- 失去所有牌的玩家出局

### 惩罚规则

**错误按铃**：
- 失去 1 张牌给每位其他玩家
- 如果只剩 1 张牌，则直接出局

#### 功能列表

| 功能 | 描述 | 优先级 | 状态 |
|------|------|--------|------|
| 用户注册 | 用户名 + 密码注册 | P0 | ✅ 已完成 |
| 用户登录 | JWT Token 认证 | P0 | ✅ 已完成 |
| 路由守卫 | 未认证重定向 | P0 | ✅ 已完成 |
| Token 管理 | 7 天有效期，自动续期 | P1 | ✅ 已完成 |
| 登出功能 | 清除 Token 和 Socket | P0 | ✅ 已完成 |

#### 注册规则

- **用户名**：
  - 长度：3-20 字符
  - 允许：字母、数字、下划线
  - 唯一性：不能重复

- **密码**：
  - 长度：6-20 字符
  - 存储：bcrypt 哈希
  - 传输：HTTPS 加密（生产环境）

#### 认证流程

```
用户输入 → 前端验证 → API 请求 → 后端验证 
→ JWT 签发 → Token 存储 → Socket 连接
```

### 常见问题

**Q: 为什么选择 lowdb 而不是 MongoDB？**  
A: 项目规模小，JSON 文件存储足够，无需额外数据库服务。

**Q: 为什么使用 Socket.IO 而不是原生 WebSocket？**  
A: Socket.IO 提供自动重连、房间管理、二进制支持等高级功能。

**Q: 前端为什么不用 TypeScript？**  
A: 保持项目简单，降低学习成本，适合快速开发。
